The 3 APIs you actually need
Polymarket exposes three separate APIs and one WebSocket layer. Most tutorials confuse them. Here's the mental model that finally made it click for me:
The four base URLs (memorize these)
python
# All Polymarket API base URLs
CLOB = "https://clob.polymarket.com" # orders + prices + your account
GAMMA = "https://gamma-api.polymarket.com" # market discovery
DATA = "https://data-api.polymarket.com" # positions, P&L, history
WS_CLOB = "wss://ws-subscriptions-clob.polymarket.com/ws/"
WS_LIVE = "wss://ws-live-data.polymarket.com" # crypto price streams

CHAIN_ID = 137 # Polygon mainnet — never use ETH mainnet
The Gamma API is read-only and doesn't need authentication. You use it to browse markets and grab their token IDs. The CLOB API is where the actual trading happens both reading the order book and placing orders. The Data API is for tracking what you've done.
For a bot you only really need two: Gamma to find markets, CLOB to trade them. The Data API is optional unless you want a dashboard.
The signature_type trap that wastes everyone 4 hours
This is the single most confusing part of the Polymarket API and I want to save you the night I lost figuring it out.
Polymarket has three different wallet types. Each requires a different signature configuration. Get this wrong and every order returns INVALID_SIGNATURE with no other clue.
If you signed up with email or Magic Link → use signature_type=1. If you connected MetaMask → use signature_type=2. The funder address is the wallet displayed on your Polymarket profile  NOT your MetaMask address.
How to find your funder address: Go to polymarket.com/settings after logging in. The address shown there is your funder. Copy it exactly. If you pass your MetaMask address by mistake, you'll get Invalid Funder Address and waste hours debugging.

Step 3 - Authenticate and derive API credentials
The Polymarket API uses a two-level auth system. L1 proves you own the wallet (one-time). L2 uses HMAC keys for every order. The client handles both.
python
import os
from dotenv import load_dotenv
from py_clob_client.client import ClobClient

load_dotenv()

client = ClobClient(
host="https://clob.polymarket.com",
key=os.getenv("PK"), # your private key
chain_id=137, # Polygon
signature_type=int(os.getenv("SIGNATURE_TYPE")),
funder=os.getenv("FUNDER") # your proxy wallet
)

# Derives + saves API credentials. Run this once.
client.set_api_creds(client.create_or_derive_api_creds())

# Verify it works:
print("Server time:", client.get_server_time())
print("Auth OK")
Run python auth.py. If you see a server time printed back -  you're authenticated. Save the api_key, secret, and passphrase that get printed; you'll need them.
Finding markets - the Gamma API trick
Before you can trade, you need token IDs. Each market has two - one for YES, one for NO. They're 78-digit strings that look like nothing. You get them from the Gamma API.
python
import requests

GAMMA = "https://gamma-api.polymarket.com"

def get_active_markets(min_volume=10000, limit=20):
"""Find markets with real liquidity, sorted by volume."""
r = requests.get(f"{GAMMA}/markets", params={
"active": True,
"closed": False,
"volume_num_min": min_volume,
"order": "volume",
"ascending": False,
"limit": limit,
})
return r.json()

for mkt in get_active_markets():
# clobTokenIds is a list — index 0 is YES, index 1 is NO
yes_id, no_id = mkt["clobTokenIds"]
print(f"{mkt['question'][:60]:60} | vol: ${mkt['volumeNum']:,.0f}")
print(f" YES token: {yes_id}")
The token ID confusion: Token IDs come from the Gamma API, not the CLOB API. Use clobTokenIds field from GET /markets. The list is usually [YES, NO] but always verify against the outcomes field. This trips up everyone the first time.
Reading the order book - before you bet
You can place an order in two ways. A market order fills immediately at whatever price is available - risky if the book is thin. A limit order rests on the book at a price you choose - safer but might not fill.
Always read the book first. Always.
python
def analyze_book(client, token_id, your_size_usdc=10):
book = client.get_order_book(token_id)
midpoint = client.get_midpoint(token_id)
best_bid = float(book.bids[0].price) if book.bids else 0
best_ask = float(book.asks[0].price) if book.asks else 1
spread = best_ask - best_bid

# How deep is the ask side at your size?
filled, cost = 0, 0
for ask in book.asks:
p, q = float(ask.price), float(ask.size)
take = min(q, your_size_usdc/p - filled)
cost += take * p
filled += take
if filled * p >= your_size_usdc: break
vwap = cost / filled if filled else 0

return {
"midpoint": midpoint, "spread": spread,
"best_ask": best_ask, "vwap_at_size": vwap,
"slippage_pct": (vwap - best_ask) / best_ask * 100
}
Why this matters: If the order book has only $50 on the ask but you try to buy $200 worth, you'll eat through 4 price levels and pay a much worse price than what you saw quoted. The VWAP function above tells you the actual price you'll pay - not the headline price.
Placing your first real order
This is the moment everything becomes real. The code below moves money. Run it once with a tiny amount first.
python
from py_clob_client.clob_types import MarketOrderArgs, OrderType
from py_clob_client.order_builder.constants import BUY, SELL

# Always check the book BEFORE placing
analysis = analyze_book(client, token_id, your_size_usdc=5.00)

if analysis["slippage_pct"] > 2.0:
print("Skipping — slippage too high")
else:
order = MarketOrderArgs(
token_id=token_id,
amount=5.00, # dollars, NOT shares
side=BUY,
order_type=OrderType.FOK # fill-or-kill
)
signed = client.create_market_order(order)
resp = client.post_order(signed, OrderType.FOK)
print(f"Order ID: {resp['orderID']}")
print(f"Status: {resp['status']}")

The complete bot  my actual main loop
This is the heart of my bot. Strategy: BTC up/down 5-minute markets. Polymarket runs short-window markets that resolve every 5 minutes based on whether BTC closed higher or lower. The bot watches Binance, decides 10 seconds before close, and places a fill-or-kill bet.
python
import asyncio, time, requests
from py_clob_client.clob_types import MarketOrderArgs, OrderType
from py_clob_client.order_builder.constants import BUY

async def trading_loop(client, starting_balance):
bal = starting_balance
day_start = bal
trades = wins = 0

while True:
# ── Kill switch — never lose more than 20% in a day ──
if bal < day_start * 0.80:
print("KILL: -20% daily loss. Stopping."); break

# ── Find the active 5-min BTC market ──
now = int(time.time())
window_start = now - (now % 300)
close_time = window_start + 300
seconds_left = close_time - now

if seconds_left > 15:
await asyncio.sleep(1); continue

# ── Get current BTC price from Binance ──
btc = float(requests.get(
"https://api.binance.com/api/v3/ticker/price?symbol=BTCUSDT"
).json()["price"])

# ── Get the strike (price at window start) ──
strike = get_strike_price(window_start)

# ── Decide direction with safety margin ──
if btc > strike * 1.0008: direction = "UP"
elif btc < strike * 0.9992: direction = "DOWN"
else: print("Skip — too close to flat"); continue

# ── Get token ID + read book ──
token_id = find_btc_market(window_start, direction)
book = analyze_book(client, token_id, 20)

# ── Skip if price already too high (no edge left) ──
if book["vwap_at_size"] > 0.85:
print("Skip — price too high"); continue

# ── Size the bet (Kelly fraction) ──
edge = 0.92 - book["vwap_at_size"] # expected win prob
size = min(bal * edge * 0.5, bal * 0.08) # half-Kelly, 8% cap
if size < 5: continue # Polymarket min

# ── Execute ──
order = MarketOrderArgs(
token_id=token_id, amount=size,
side=BUY, order_type=OrderType.FOK
)
signed = client.create_market_order(order)
resp = client.post_order(signed, OrderType.FOK)
trades += 1
print(f"#{trades} {direction} ${size:.2f} → {resp['status']}")

# ── Wait for resolution ──
await asyncio.sleep(320)
# Then update balance from Data API...
The two safety lines that saved my account: if bal < day_start * 0.80: break (kill switch) and if size < 5: continue (minimum bet check). Without the kill switch I would have lost everything on day 4 when a flash drop killed three trades in a row. Without the minimum check the API would have rejected every order silently.

The mistakes - what cost me

Day 4 - flash drop wiped 9% in 18 minutes
BTC dropped $400 right before window close. Three trades in a row went the wrong way. Kill switch fired and stopped the bot until next day. Lesson: trust the kill switch.
Day 7 - slippage on a thin market
Tried to enter $40 on a market with $60 on the ask. Got filled across 3 price levels. Paid 4% more than I expected. Lesson: always check VWAP at your actual size.
Day 11 - RPC rate limit
Free Polygon RPC started returning errors. Bot missed 6 entry windows. Switched to Alchemy paid tier ($49/month). Zero misses since. Lesson: don't cheap out on infrastructure.
Day 16 - order rejected for 0.001 size error
Calculated bet size as 4.999, Polymarket minimum is 5.00. Order silently rejected. Now I round up to nearest $1 above the floor. Lesson: pad your minimums.

I'm not asking you to take my word for any of this. Every line of code in this article is built on official, public, MIT-licensed repositories. Bookmark these - they're the foundation of every serious Polymarket bot in production right now.
Official Polymarket repositories
1.Polymarket / py-clob-client
Official Python SDK. Order placement, signing, authentication. Every code block in this article uses this.
2. Polymarket / py-clob-client-v2
Next-gen SDK with cleaner API and explicit L1/L2 auth. Use this for new projects.
3. Polymarket / clob-client
TypeScript / JavaScript SDK. Same functionality as py-clob-client for Node.js bots.
4. Polymarket / rs-clob-client
Rust SDK for low-latency execution. Used by the wallets that win the latency race.
5. Polymarket / agents
Polymarket's own AI trading agents framework. Langchain + Chroma + LLM-driven autonomous trading.
6. Polymarket / examples
Code examples for Proxy/Safe wallets, splits, merges, redemptions. Read this before debugging signature errors.
7. Polymarket / polymarket-cli
Command-line tool for browsing markets, checking balances, placing orders. Great for debugging without writing code.
8. Polymarket / magic-proxy-builder-example
Next.js example for trading from email/Magic accounts. Essential if you signed up with email instead of MetaMask.
9. Polymarket / clob-order-utils
Low-level utilities for building and signing orders. You only need this if you're building a custom signer.
10. Polymarket / ctf-exchange-v2
The actual on-chain exchange contracts. Read the source if you want to understand what your orders settle into.
Community repos worth bookmarking
1.HuakunShen / polymarket-kit

Unified Python wrapper around all four APIs (CLOB, Gamma, Data, WebSocket) with Pydantic validation.
